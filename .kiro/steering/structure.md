# Project Structure

```
├── src/                    # Library source (CDK constructs) — compiled by jsii to lib/
│   └── index.ts            # Exports BunFunction and BunLambdaLayer constructs
├── lib/                    # Compiled output (generated, do not edit)
│   └── bun-lambda-layer-*.zip  # Pre-built Lambda layer artifact
├── command/                # Build tooling scripts (run with Bun directly)
│   ├── buildLayer.ts       # Downloads Bun binary, compiles runtime, creates layer zip
│   └── runtime.mts         # The actual Lambda custom runtime (deployed inside the layer)
├── test/                   # Test files (run with `bun test`)
│   └── setup.test.ts       # Test infrastructure validation
├── dist/                   # Release packaging output (generated)
├── docs/                   # Additional documentation
├── .projenrc.ts            # Projen project definition (source of truth for project config)
├── .projen/                # Projen-generated metadata (do not edit)
├── .github/workflows/      # CI/CD pipelines (build, release, version checks)
├── .kiro/                  # Kiro AI assistant configuration
│   ├── steering/           # Steering rules for AI context
│   └── specs/              # Feature specifications
├── tsconfig.json           # jsii TypeScript config (src → lib)
├── tsconfig.dev.json       # Dev TypeScript config (includes test, .projenrc.ts)
└── package.json            # Generated by projen — do not edit directly
```

## Key Conventions

- **`src/`** contains only the public construct library API. It is compiled by jsii and must follow jsii rules (no enums with computed values, explicit types on public APIs, etc.).
- **`command/`** contains Bun-native scripts that are NOT part of the published package. They use Bun APIs directly (Bun.build, Bun.spawnSync, Glob).
- **`test/`** uses `bun:test` as the test runner and `fast-check` for property-based tests.
- **`lib/`** is the jsii compilation output AND contains the pre-built layer zip. Both are committed to git.
- Files in the project root (package.json, tsconfig.json, .eslintrc.json, etc.) are mostly projen-managed — edit `.projenrc.ts` and run `bunx projen` to regenerate.
