# Project Structure

```
├── src/                    # Library source (CDK constructs)
│   └── index.ts            # Exports BunFunction and BunLambdaLayer constructs
├── command/                # Build tooling scripts (run with Bun directly)
│   ├── build.ts            # Full build orchestration (layer + bundle + declarations)
│   ├── buildLayer.ts       # Downloads Bun binary, compiles runtime, creates layer zip
│   └── runtime.mts         # The actual Lambda custom runtime (deployed inside the layer)
├── test/                   # Test files (run with `bun test`)
│   ├── constructs.test.ts  # CDK construct unit tests
│   ├── buildLayer.test.ts  # Build layer script tests
│   ├── runtime.test.ts     # Runtime loop integration tests
│   ├── setup.test.ts       # Test infrastructure validation
│   └── integration/        # AWS integration tests (deploy + destroy)
├── dist/                   # Build output (generated, gitignored)
├── lib/                    # Layer zip output from buildLayer.ts (generated, gitignored)
├── docs/                   # Additional documentation
├── examples/               # Example CDK app for integration testing
├── .github/workflows/      # CI/CD pipelines (build, release, integration, version checks)
├── .kiro/                  # Kiro AI assistant configuration
│   └── steering/           # Steering rules for AI context
├── .oxlintrc.json          # Oxlint config (extends @beesolve/lint-config preset)
├── .oxfmtrc.json           # Oxfmt formatter config
├── tsconfig.json           # TypeScript config (declaration emit only)
├── package.json            # Hand-written package config
└── .bun-version            # Pinned Bun version for CI
```

## Key Conventions

- **`src/`** contains only the public construct library API. It is bundled to ESM with `Bun.build` and declarations generated by `tsc`.
- **`command/`** contains Bun-native scripts that are NOT part of the published package. They use Bun APIs directly (Bun.build, Bun.spawnSync, Glob).
- **`test/`** uses `bun:test` as the test runner and `fast-check` for property-based tests.
- **`dist/`** is the build output (JS bundle, declarations, layer zip). Gitignored — built fresh in CI.
- **`lib/`** is the intermediate layer zip output. Gitignored — built fresh in CI.
- All project config (package.json, tsconfig.json, lint/format configs) is hand-written — no code generation.
